Research

Questions

  1. What is the definition of statistics?
  2. What is a dataset? From the observation units to the dataset. Attributes and variables.
  3. Find interesting application of statistics in Cybersecurity.

Answers

  1. Statistics is the science that, thanks also to methods and mathematics tools, deals with the information acquisition about a phenomenon and how these information are organized, analysed and interpreted.

  2. A dataset is a collection of descriptions of different instances of the same phenomenon. These descriptions could take a variety of forms, but it is important that they are descriptions of the same thing. For example, my grandfather collected the daily rainfall in his garder for many years; we could collect the height of each person in a room.
     An observation unit, sometimes also called statistical unit, is the single inhabitans of the population over which is performed an observation. In other words, the unit of observation is the unit at or for which data is collected. At the end, after choosing a phenomenon to observe, choosing observation units and collecting descriptions, we have a dataset.

    An attribute is a quality of an object (person, thing, etc.). As an example for a person, the possible attributes are: gender, age, sports etc. A variable is the set of all possible value that an attribute can take.

    • A very common way in which Statistics is used in Cybersecurity is to assess trends in order to take countermeasures. As an example, in the following link is showed how much UK organisations deal with Cybersecurity; the reasearch has been done by the UK government
      https://assets.publishing.service.gov.uk/government/uploads/system/uploads/attachment_data/file/987652/CSBS_2021_business_trends_infographic.pdf
    • The purpose of the following paper is to describe the development of the cybersecurity datasets used to train the algorithms which are used for building IDS detection models, as well as analyzing and summarizing the different and famous Internet of things (IoT) attacks.
      https://www.researchgate.net/profile/Anton-Konev-2/publication/358223470_The_Comparison_of_Cybersecurity_Datasets/links/61f6d421007fb50447260c47/The-Comparison-of-Cybersecurity-Datasets.pdf
    • Another important way in which Statistics is used in Cybersecurity, is to discover if something dangerous is happening in our incoming network. The following paper illustrates a statistical analysis of network data. https://www.researchgate.net/publication/261581408_Statistical_Analysis_of_Network_Data_for_Cybersecurity

Questions

  1. Explain the concept of Distribution.
  2. Explain the concept of Univariate and Multivariate distribution.
  3. Explain the concept of Conditional and Marginal distributions.

Answers

  1. In statistics, a Distribution is a representation of the way in which the different value of a variable (of an attribute) are distributed in the statistical units of the statistical population.
    As an example let's consider an interview done in a cultural centre to establish the number of subscribers that speak a certain language. The following is the distribution gained from the interview:
    English: 118
    Italian: 200
    Spanish: 68

  2. We define Univariate distribution of an attribute, the listing of the number of times each value of the variable associated to the attribute appears.

    A Multivariate distribution is a Univariate distribution on which the listing is performed over more than one attribute.

  3. A Conditional distribution is the distribution of an attribute to which is associated another attribute. Alternatively, it is a simple distribution gained from a double distribution. Given a table in which we examine simultaneously the variables of two attributes X and Y relatives to the same statistical population, we define Conditional distribution each of the simple distributions of data expressed from a row or from a column of the table itself.

    Given a table that organize the values of two variable of two attribute. A Marginal distribution is each of the simple distributions associated to only one attribute, regardless to the other. So, given the conjoint distribution represented from the abovementioned table, we have two margianl distributions: one for the row and for the column, expressed from the total of the rows and from the total of the columns; these distributions are often reported at the margin of the table.

Questions

  1. Summarize the main aspects the of the CSV protocol RFC 4180 (definition and rules).

Answers

  1. A comma-separated values (CSV) file is a text file that uses a comma to separate values. A CSV file stores tabular data, such as numbers or text in a spreadsheet, in plain text. Each line of the file is a data record. Each record consists of one or more fields, separated by commas. Thus, the file format gets its name from using the comma to separate different values. Example where each record is separated by a new line (CRLF):
    value1,value2,value3CRLF
    value4,value5,value6CRLF
     There maybe an optional header line appearing as the first line of the file with the same format as normal record lines. This header will contain names corresponding to the fields in the file and should contain the same number of fields as the records in the rest of the file. Each field may or may not be enclosed in double quotes.
    Example:
    "age","heigth"CRLF
    10,130CRLF
    51,180

Questions

  1. Illustrate the concept of conditional, joint, marginal (relative) frequency using a simple bivariate distribution.
  2. Illustrate the concept of statistical independence and the resulting mathematical relationships between the above frequencies.

Answers

  1. Given a two-way frequency table, conditional realtive frequency is a fraction that tells you how many members of a group have a particular characteristic. More technically, it is the ratio of a frequency in the center of the table to the frequency’s row total or column total (and not the number that shows the total number of statistical unit tipically at the bottom-right).
    As an example, let's consider the following table where the number of successful throws into a basket are counted:

    Two-way Realtive Frequency Table
    Boys Girls Total
    Made 7 6 13
    Missed 11 9 20
    Total 18 15 33
    the conditional relative frequency that, if you are given a girl, she also missed the basket is 9/15.

    With joint realtive frequency we mean the realtive frequencies inside the two-way frequency table.
    Given the following table where the number of successful throw into a basket are counted:

    Two-way Realtive Frequency Table
    Boys Girls Total
    Made 7 6 13
    Missed 11 9 20
    Total 18 15 33
    the joint relative frequency of boys who missed the basket is 11/33.

    The marginal relative frequency is the opposite of the joint one because the lateral row/column of the table is considered.
    Given the following table where the number of number of successful throw inside a basket are counted:

    Two-way Realtive Frequency Table
    Boys Girls Total
    Made 7 6 13
    Missed 11 9 20
    Total 18 15 33
    the marginal relative frequency of girls who played the game is 15/33.

  2. Given two events A and B, they are statistical independent if and only if event A occurring confers no change to the probability of B, when compared to its unconditional probability.

    The mathematical relationship with conditional relative frequency is that, since conditional relative frequency holds when two non-independent event happen, the conditional realtive frequency is not independent.

    For the other kinds of frequencies, the methematical relationship is simply the transportation to the fact that a certain event occours. Taking the previous example, the probability to pick a girl who played the game is 45.5% and the probability to pick a boy who missed the basket is 21.2%.

Research on application

Questions

  1. A survey on ONLINE algorithms (Welford).
  2. In particular, Knuth recursion for the computation of the arithmetic mean (average).

Answers

  1. An online algorithm is an algorithm where inputs are arriving over time, and we need to make decisions on the fly without knowing what will happen in the future. An offline algorithm, instead, requires all information BEFORE the algorithm start.
     Selection sort is offline because step 1 is Find the minimum value in the list. To do this, you need to have the entire list available - otherwise, how could you possibly know what the minimum value is? You cannot. Insertion sort, by contrast, is online because it does not need to know anything about what values it will sort and the information is requested WHILE the algorithm is running. Simply put, it can grab new values at every iteration.
  2. The average is typical calculated doing: sum/#of_elements. Another way to calculate the average is by doing the following:
    previous_average + [(newer_number - previous_average) / #of_elements]
     As an example, knowing that the average of two numbers is 7.5, if the average with the new number 3 has to be calculated, we can simply do:
    7.5 + [(3 - 7.5) / 3] = 7.5 + [-4.5 / 3] = 6.
    Because the average can be calculated when a new number is added, this algorithm is considered to be an online algorithm.
    With this latter formula we have several advantages:
    1. the accumulator which holds the sum of the numbers is not needed (hence we do not have to store a probably very big number)
    2. avoid dealing with operataions that use a very big number like the probably one of the sum
    For these reason it is very often used in information technology programs.
